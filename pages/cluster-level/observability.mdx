## Get Started PipeOps Cluster Observability 

This guide will walk you through the process of accessing and exploring the metrics for your deployed Clusters. With PipeOps, you can gain valuable insights into the performance and behavior of your Clusters. Let's get started!

### Accessing Cluster Metrics
To view the metrics for your Cluster, follow these steps:

Navigate to your Cluster in PipeOps.
Click on the "Observability" tab.
You will be presented with an overview of the Cluster metrics, providing a high-level summary of its performance.

![Guide image](/assets/cluster_chrome_yjky9fqpga.png)

The image above illustrates what the Cluster metrics overview might look like. It gives you a quick glimpse of how your Cluster is performing in terms of various metrics.


### Exploring Individual Metrics

PipeOps allows you to dive deeper into the details of each metric. Let's explore some of the key metrics available:

### CPU Metric

The CPU metric provides insights into the CPU usage of your Cluster. It helps you understand the processing power consumed by your application. Here's an example of how the CPU metric might be visualized:

![Guide image](/assets/cluster_cpu.png)

### Memory Metric

The memory metric provides information about the memory usage of your Cluster. It gives you visibility into how much memory your application is utilizing. Here's an example of how the memory metric might be represented:

![Guide image](/assets/cluster_memory.png)

### Storage Metric

The storage metric focuses on the storage usage of your Cluster. It helps you track the amount of disk space your application is occupying. Here's an example of how the storage metric might be displayed:

![Guide image](/assets/cluster_storage.png)

### Network I/O Metric 

The network I/O metric provides insights into the input and output activity of your Cluster. It gives you an understanding of the data transfer happening between your application and external systems. Here's an example of how the network I/O metric might be visualized:

![Guide image](/assets/cluster_network.png)

### Control Plane Metric 

The control plane metric provides insights into the performance of your cluster control plane. It helps you monitor the health and efficiency of the control plane components that manage the cluster. Here's an example of how it might be visualized:

![Guide image](/assets/cluster_contro_plane.png)

Monitoring the control plane metric allows you to identify potential issues and optimize cluster management. Keep an eye on this metric to ensure smooth operation and maximize performance.

By exploring these individual metrics, you can gain a comprehensive understanding of how your Cluster is performing in different areas.

With PipeOps Cluster observability, you have the power to monitor and analyze your Cluster's metrics effectively. Leverage these insights to optimize performance, identify bottlenecks, and make data-driven decisions.

Now that you're equipped with the knowledge of how to access and explore Cluster metrics in PipeOps, you're ready to unlock the full potential of monitoring and optimizing your Clusters. Happy analyzing!
